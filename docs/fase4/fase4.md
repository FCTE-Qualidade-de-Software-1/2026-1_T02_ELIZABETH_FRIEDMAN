# Fase 4 - Introdução

Nesta etapa do projeto, são apresentados os resultados da execução do plano de avaliação definido na [Fase 3](https://fcte-qualidade-de-software-1.github.io/2026-1_T02_ELIZABETH_FRIEDMAN/fase3/fase3/), bem como a análise dos dados coletados para as características de **Adequação Funcional** **Confiabilidade** do sistema AGIO. Os resultados obtidos são comparados aos critérios e hipóteses estabelecidos na [Fase 2](https://fcte-qualidade-de-software-1.github.io/2026-1_T02_ELIZABETH_FRIEDMAN/fase2/fase2/), permitindo verificar o atendimento dos requisitos de qualidade definidos para o sistema.

A partir das métricas coletadas durante os testes, busca-se avaliar o comportamento do AGIO em relação à ocorrência de falhas, ao sucesso na execução das operações, ao tratamento de entradas inválidas e à proteção contra acessos indevidos. A análise dos resultados possibilita identificar pontos fortes, limitações e oportunidades de melhoria relacionadas à confiabilidade do sistema.

## Metodologia

A interpretação dos resultados foi realizada com base na abordagem Goal-Question-Metric (GQM), seguindo as etapas descritas a seguir:

### Validação das Hipóteses

Para cada questão definida na [Fase 2](https://fcte-qualidade-de-software-1.github.io/2026-1_T02_ELIZABETH_FRIEDMAN/fase2/fase2/), os valores obtidos por meio das métricas implementadas e executadas conforme o plano de avaliação descrito na [Fase 3](https://fcte-qualidade-de-software-1.github.io/2026-1_T02_ELIZABETH_FRIEDMAN/fase3/fase3/) foram comparados com os respectivos critérios de aceitação estabelecidos previamente.
### Julgamento dos Resultados

As hipóteses foram consideradas:

* **Confirmadas**, quando os resultados alcançaram ou superaram os critérios definidos
* **Refutadas**, quando os resultados ficaram abaixo dos limites estabelecidos

### Análise das Evidências

Os resultados quantitativos foram complementados pela análise das evidências registradas durante a execução dos testes. Entre as evidências consideradas estão:

- Logs da aplicação e do servidor
- Mensagens de erro apresentadas pelo sistema
- Registros das operações executadas
- Comportamentos observados durante os cenários de teste
- Capturas de tela e demais registros produzidos durante a avaliação

Essa etapa teve como objetivo identificar fatores que possam ter influenciado os resultados obtidos, bem como compreender as causas associadas às falhas ou aos comportamentos adequados observados.

### Interpretação dos Resultados

Com base nos dados coletados e nas evidências analisadas, foi realizada uma interpretação dos resultados para cada métrica avaliada. Essa interpretação permitiu identificar o grau de atendimento dos requisitos de confiabilidade, bem como os principais pontos fortes e limitações do sistema.

### Conclusão

Por fim, os resultados das métricas e a análise das evidências foram consolidados para produzir conclusões sobre a confiabilidade do AGIO. Essas conclusões servem de base para a elaboração de recomendações de melhoria e para a avaliação do atendimento dos objetivos de qualidade definidos para o sistema.
